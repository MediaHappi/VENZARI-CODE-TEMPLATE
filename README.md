## What is this?

This is the companion repository for **[PROJECT NAME]**. It is the persistent memory, task system, and governance layer for your engineering team.

Every task, decision, and session is tracked here. VENZARI CODE reads this repo to understand your environment before every session.

## Quick Start

```bash
# Start a governed session on this repo
venzari-code start /path/to/this/repo

# Discover what VENZARI CODE knows about this repo
venzari-code discover /path/to/this/repo

# Claim a task
venzari-code claim <task-id> --role implement

# Complete a task (with evidence)
venzari-code complete <task-id> --summary "What was done" --evidence "Tests pass, N events" --skill test
```

## Sessions

Start every session by reading:
1. `SESSION_STARTUP.md` — what to do at the start
2. `CONTEXT.md` — current project state
3. `system-map/CURRENT_STATE.md` — live running services

Then run: `venzari-code start .`

## Tasks

Tasks live in `.tasks/` as JSON files. Format:

```json
{
  "id": "PROJ-001-feature-name",
  "title": "Short description",
  "priority": "high",
  "description": "What needs to be done and why",
  "status": "pending"
}
```

## Governance

VENZARI CODE enforces:
- **Mode transitions** — discover → plan → implement → validate → review → commit
- **Tool guard** — only permitted tools in each mode
- **Evidence requirements** — minimum 3 tool calls recorded before task completion
- **Loop detection** — prevents infinite tool call loops
- **Secret scanner** — blocks API keys in commits
